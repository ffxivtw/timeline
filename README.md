# FFXIV 跨伺服器改版時間軸

對照 Final Fantasy XIV 各區域服務（**國際服 / 中服 / 韓服 / 繁中服**）的版本上線時間，並依各服相對國際服的延遲規律，**預估落後服下個版本的上線日**。「現在」以瀏覽器本地時間標示。

## 功能

- **下個版本預估**：選擇目標伺服器（預設繁中服），顯示目前版本、預估的下個版本與上線日，並透明列出推估依據（近幾版平均延遲天數、信心等級）。
- **版本時間軸**：每服一軌的橫向時間軸，含即時「現在」線，各服預估下個版本以空心點標示。
- **版本對照表**：版本 × 伺服器的上線日期一覽。

## 開發

```bash
npm install
npm run dev      # 開發伺服器
npm run test     # 執行單元測試（預估邏輯）
npm run build    # 型別檢查 + 打包到 dist/
npm run preview  # 預覽打包結果
```

## 維護版本資料

所有資料集中在 [`src/data/versions.ts`](src/data/versions.ts)，手動維護、無自動抓取來源。

- `global`（國際服）為所有預估的**基準**，日期為實際上線日。
- `china` / `korea` / `tw` 目前的日期為**示範用估算值**（依固定延遲推算）。
  請以各服官方公告的實際上線日期替換。

新增版本時，於 `versions` 陣列加入一筆，未上線的伺服器留空即可；程式會自動依國際服日期排序。

## 部署

打包設定 `base: './'`（相對路徑），可直接部署到 GitHub Pages 專案頁。
專案內含 `.github/workflows/deploy.yml`，推到 `main` 分支即自動建置並發佈至 GitHub Pages
（需在 repo 的 Settings → Pages → Source 選擇 **GitHub Actions**）。
